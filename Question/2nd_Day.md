### Day-2

**Task Title:**
Product Price Update & Personal Profile Page

---

## Objective

Focus on **meaning and structure**, not design.

---

## Task Requirements

### 1. HTML Boilerplate (Required)

Your file must include:

- `<!DOCTYPE html>`
- `<html>`
- `<head>`
  - `<meta charset="UTF-8">`
  - `<title>`
  - `<link>` (for favicon)

- `<body>`

---

### 2. Text and Basic Formatting

Inside `<body>`:

- One `<h1>` as the main title
- At least two headings from `<h2>` to `<h6>`
- Two paragraphs using `<p>`
- One `<hr>`
- One `<br>` inside a paragraph
- Use `<b>`, `<i>`, and `<u>` at least once

---

### 3. Lists, Links, and Image

- One unordered list (`<ul>`) with 3 items
- One ordered list (`<ol>`) with 3 items
- One link using `<a>`
- One image using `<img>` with `src` and `alt`

---

### 4. Form

Create a form containing:

- One text input (username)
- One password input
- A `<label>` for each input
- One `<button>` to submit

---

### 5. Semantic Formatting

Use these tags correctly:

- `<strong>` for important text
- `<em>` for emphasized text
- `<mark>` to highlight a word
- `<del>` for old price and `<ins>` for new price
- `<sub>` in a chemical formula (example: H₂O)
- `<sup>` in a scientific or mathematical formula (example: E = mc²)

---

### 6. Quotations and Citations

Add:

- One long quote using `<blockquote>`
- One short quote using `<q>` inside a sentence
- One abbreviation using `<abbr>` with a `title` attribute
- One `<address>` section containing:
  - Name
  - Email
  - Location or City

---

### 7. Invisible Helpers

- Add at least two HTML comments explaining sections
- Add a favicon using `<link>` inside the `<head>`

---

### 8. Table (Very Important)

Create a table titled **Product Price Table** using:

- `<table>`
- `<tr>`
- `<th>`
- `<td>`

Table columns:

1. Product Name
2. Old Price
3. New Price
4. Status (Updated / Unchanged)

Include at least 3 data rows.

---

## Rules

- Use only Day-1 HTML tags
- No CSS
- No JavaScript
- No extra HTML tags not taught

---

### Hint

Think about what the text **means**, not how it looks.
